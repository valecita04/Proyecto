Github

GitHub es una forja (plataforma de desarrollo colaborativo) para alojar proyectos utilizando el sistema de control de versiones Git. Se utiliza principalmente para la creación de código fuente de programas de ordenador. El software que opera GitHub fue escrito en Ruby on Rails. Desde enero de 2010, GitHub opera bajo el nombre de GitHub, Inc. Anteriormente era conocida como Logical Awesome LLC. El código de los proyectos alojados en GitHub se almacena generalmente de forma pública.

¿Qué es GitHub y para qué sirve? GitHub es una plataforma basada en la nube donde puedes almacenar, compartir y trabajar junto con otros usuarios para escribir código. Almacenar tu código en un "repositorio" en GitHub te permite lo siguiente: Presentar o compartir el trabajo. Seguir y administrar los cambios en el código a lo largo del tiempo

¿Qué puedes hacer en GitHub? GitHub es una gran comunidad de código abierto. Puedes explorar GitHub para encontrar repositorios, temas, código, usuarios y organizaciones interesantes que pueden inspirar tu propio trabajo o complementar tu aprendizaje.

Características principales de GitHub:

Repositorios: Son espacios donde se guarda el código de un proyecto, junto con su historial de versiones y cambios.

Git: GitHub utiliza el sistema de control de versiones Git, que permite rastrear los cambios en el código a lo largo del tiempo.

Colaboración: Facilita la colaboración entre desarrolladores, permitiéndoles compartir, revisar y fusionar código.

Ramas (Branches): Permiten crear versiones paralelas del código para trabajar en nuevas características o correcciones sin afectar el proyecto principal.

Solicitudes de extracción (Pull Requests): Permiten a los desarrolladores proponer cambios a otros usuarios, que pueden revisarlos y fusionarlos.

GitHub Pages: Posibilidad de alojar sitios web estáticos desde los repositorios de GitHub.

Integración con otros servicios: Permite integrar GitHub con otras herramientas de desarrollo, como VS Code, Jenkins, etc.

Codespaces: Entornos de desarrollo en la nube para trabajar en proyectos desde cualquier lugar.

Actions: Herramientas para automatizar tareas de desarrollo, como pruebas e implementación.

GitHub Actions: Automatiza el flujo de trabajo de desarrollo, incluyendo pruebas, implementación y otros procesos.

Paquetes de GitHub: Permite alojar paquetes y código en un solo lugar.

GitHub Packages: Permite alojar paquetes y código en un solo lugar.

Control de acceso: Permite gestionar los permisos de acceso a los repositorios.

Seguimiento de problemas (Issues): Permite registrar y gestionar problemas y tareas relacionadas con los proyectos.

Gráficos: Ofrece gráficos de progreso del proyecto, contribuciones de los desarrolladores, etc.
